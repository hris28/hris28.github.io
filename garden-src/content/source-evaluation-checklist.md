# Source Evaluation Checklist

Security and technology writing should be verifiable. Before trusting or citing a source, ask:

| Check | Question |
|---|---|
| Authority | Who created the source? |
| Evidence | Does it cite data, documentation, code, or primary sources? |
| Date | Is the information current enough for the topic? |
| Scope | Is it making a narrow claim or an exaggerated one? |
| Incentive | Is the source selling something or promoting a product? |
| Reproducibility | Can others test or verify the claim? |
| Consensus | Do other credible sources agree? |

## Better sources for security writing

Prefer:

- official documentation
- standards bodies
- academic papers
- reputable security advisories
- vendor documentation for product-specific claims
- primary legal or policy documents
- transparent technical writeups

Be cautious with:

- unsourced social media posts
- SEO-driven listicles
- outdated tutorials
- screenshots without context
- claims that imply certainty without evidence

## Citation habit

When writing an article, include a short **References** section. Even a few reliable references make the page more trustworthy.
